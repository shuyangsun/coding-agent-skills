## What was built

`wave6_query.py` + runner hooks, all query-time only (every arm shares its base
collection; the 0020 twin-baseline jitter control runs in every benchmark):

1. **Router features (step 1):** deterministic query-text features (identifier/path
   density, multipart markers — gate transforms), and **first-pass retrieval-state
   confidence features** (dense∩sparse top-10 file agreement `agree10`, fused-head
   concentration, dense top score `d_top` and gap `d_gap`) logged per query into a new
   `route.*` TSV channel.
2. **Self-query soft filters (step 2):** high-precision lang/kind/path extraction
   (fires on 17% of gold queries), applied as an extra _rank-fused list_ (weighted RRF,
   w=0.5) — never an additive boost (the Wave-5 lesson).
3. **Decomposition (step 3):** Nemotron subqueries (or a deterministic
   sentence/conjunction splitter as the LLM-free control), each subquery retrieved
   through the full hybrid path, lists fused by weighted RRF (orig 1.0 / sub 0.5).
4. **HyDE / multi-query (step 4):** HyDE swaps only the **dense** query vector (sparse
   keeps the original query to protect exact identifiers); multi-query fuses 3
   paraphrase lists. Gates: `all` vs identifier-protective (`no-ident`) /
   `multipart`.
5. **CRAG-style confidence gate (step 5):** rerank only when the first-pass state says
   the reranker is likely to help; thresholds fitted on dev by `wave6_fit_gate.py`
   (exhaustive single-feature + OR-pair search over logged features) and frozen before
   any held-out run. Plus a `protect_top1` rerank guard (restore the fused top chunk
   unless the reranker overrules by a margin).
6. **Qwen3-Reranker-4B serving (the A/B):** vLLM `--runner pooling` with
   `hf_overrides={"architectures":["Qwen3ForSequenceClassification"],
"classifier_from_token":["no","yes"],"is_original_qwen3_reranker":true}` and the
   official instruction template wrapped around query/docs in the runner
   (`rerank.template="qwen3"`).

## Why a gate looked promising — the 0020 mining

Joining `0020`'s per-query rows (`w4llm-rrk` − `w4-ctx-llm-gemma4-all`, all 207
queries): the bge-m3 reranker **helps weak first-pass queries and hurts strong ones** —
helped queries average base pMRR 0.330, hurt ones 0.689, and **37/70 hurt queries had
the primary already at rank 1** before rerank demoted it. Query-TEXT features do not
separate the populations at all (identical length/identifier stats) — but an **oracle
per-query gate** would have been worth held-out **nDCG +0.057 / pMRR +0.075** while
keeping the coverage gains. That motivated retrieval-STATE confidence features.

## Result 1 — the fitted confidence gate: aggregate-positive, slice-rejected

Dev fit (n=96): oracle +0.083 nDCG / +0.107 pMRR at fire-rate 0.38; best frozen rule
`d_top≤0.752 OR d_gap≤0.0215` (fires 38% — rerank when the dense head is weak or flat;
a second, counter-intuitive winner `agree10≥5` says rerank helps when dense and sparse
**agree**, i.e. when there is a good pool to arbitrate). Live dev confirm: **nDCG
+0.053, pMRR +0.070** vs same-draw base.

Held-out (n=111), record draw, vs `w4-ctx-llm-gemma4-all` (0.759/0.693/0.913/0.847;
twin +0.002…+0.009; replicate-draw twin spread reached ±0.014 — wider than 0020's
±0.005 on this corpus rev):

| arm                                                 | R@20  | nDCG@10    | pMRR   | sentCov    | hit@5  | repeated regressed cells (2 draws) |
| --------------------------------------------------- | ----- | ---------- | ------ | ---------- | ------ | ---------------------------------- |
| `w4llm-rrk` (bge always — 0020's lever, reproduced) | 0.995 | **−0.045** | −0.048 | +0.015     | +0.045 | 28                                 |
| `w6-rrk-gate` (38% fire)                            | 0.995 | +0.008     | +0.007 | **+0.033** | +0.036 | **10**                             |
| `w6-rrk-gate-cov` (75% fire)                        | 0.995 | −0.025     | −0.020 | +0.011     | +0.027 | 29                                 |

The gate **dominates always-rerank on every held-out aggregate** (it even beats it on
sentinel 0.946 vs 0.928 — reranking _confident_ queries was pushing sentinel chunks out
of the pack) and is identical across both draws (rerank scores are deterministic). But
the dev ranking lift collapsed held-out (+0.053 → +0.008: the d_top/d_gap thresholds
partially overfit dev), and the **repeated-cells test fails it**: 10 cells regress in
BOTH independent draws, several with identical magnitudes (hard×nl pMRR −0.095/−0.095,
coding-agent-skills-nl −0.050/−0.050, alpha-zero-code −0.109/−0.155) — 3–5× the twin's
worst repeated cell (−0.033). Mechanism: **adverse selection** — alpha-zero is where
the bge reranker is most destructive _and_ where first-pass confidence is lowest, so
the gate routes to the reranker exactly the queries it then damages. **REJECT as a
default.** (It remains the better operating point than always-bge if Wave 7 wants that
lever — but Result 3 makes the whole question moot.)

`protect_top1` guards (margin sweep 0.05–0.2, `min_top_score`): best dev arm +0.007
nDCG, non-monotonic in the margin (noise), redundant once gating works — **REJECT**.

Two diagnostics worth keeping: (a) Nemotron's HyDE passages **invent plausible
identifiers** (e.g. a `source_transcript_id` field that exists nowhere in the corpus) —
exactly the dilution risk the plan predicted, visible here as the sentinel-coverage
cost; (b) transform arms raise the **contamination-exposure** diagnostic (≈1.2 → 1.4–1.5
flagged synthesis-doc paths in top-20): LLM-written queries/passages drift toward the
benchmark/plan vocabulary of this repo's own docs.

## Result 3 — the reranker A/B: serving fidelity was the whole story

Score sanity (the plan's Wave-3 gate, finally executed): with the official instruction
template, Qwen3-Reranker-4B separates a relevant / near-topic / off-topic triple as
**0.9994 / 0.0092 / 0.0000**; **without the template the same model's scores are
inverted** (relevant 0.137 < off-topic 0.907). A naive cross-encoder serving (what
Infinity would have done in Wave 3) would not merely have underperformed — it would
have anti-ranked. This is why the "SOTA pick" sat unmeasured.

Held-out (n=111), always-on, ungated, vs `w4-ctx-llm-gemma4-all`:

| arm                             | R@20  | nDCG@10            | pMRR               | sentCov            | hit@5              | ms/q |
| ------------------------------- | ----- | ------------------ | ------------------ | ------------------ | ------------------ | ---- |
| base `w4-ctx-llm-gemma4-all`    | 0.995 | 0.759              | 0.693              | 0.913              | 0.847              | 38   |
| `w4llm-rrk` (bge-m3)            | 0.995 | 0.714              | 0.644              | 0.928              | 0.892              | 147  |
| **`w6-rrk-qwen4b`**             | 0.995 | **0.848 (+0.090)** | **0.800 (+0.107)** | **0.955 (+0.042)** | **0.991 (+0.144)** | 555  |
| `w6-rrk-qwen4b-gate`            | 0.995 | 0.825 (+0.067)     | 0.770 (+0.078)     | 0.952              | 0.928              | 275  |
| replicate draw: `w6-rrk-qwen4b` | 0.995 | 0.845              | 0.795              | 0.955              | 0.991              | 555  |

- **Largest single-variable win of the campaign** (vs headers +0.056, contextual
  retrieval +0.031), replicated to the third decimal. Code domain: nDCG **+0.119**,
  pMRR **+0.139**, hit@5 **+0.192** (0.986). NL domain positive too (+0.034/+0.046).
- It **exceeds the bge-gate oracle** (dev 0.828 vs oracle-of-gating-bge 0.747): the
  reranker quality ceiling, not routing, was the binding constraint. The gate now only
  _caps_ the win (+0.067 vs +0.090) — useful solely as a latency knob (275 vs 555 ms).
- `protect_top1` guards on qwen: dev **−0.044** — most of qwen's value IS overruling
  wrong fused top-1s. **REJECT.**
- Slice picture (both draws): **every** repo / domain / category / difficulty /
  multiplicity single-axis slice positive on nDCG except two named caveats —
  **alpha-zero pMRR** (−0.008 record / −0.067 replicate; its sentinel +0.088 and hit@5
  +0.176 improve) and **category=session-prompting** (nDCG −0.012 / pMRR −0.018,
  identical in both draws; its hit@5 +0.105 improves). Crossed repo×domain cells show 7
  repeated regressions (hard×nl −0.103, alpha-zero-api-nl −0.123, website-nl −0.046):
  the reranker deterministically prefers a near-duplicate doc over the pinned primary
  for a handful of nl queries whose pack coverage nonetheless improves.
- **Decision: PROMOTE as the campaign reranker, always-on** (Apache-2.0, so
  license-clean, but GPU + vLLM serving → campaign, not portable). The two caveat
  slices are exactly answer-stage questions — rank-1 precision vs pack coverage — and
  go to Wave 7 with named query populations. The 0020 "coverage-vs-precision lever"
  framing is **retired**: it was a property of the bge-m3/MiniLM quality tier, not of
  reranking.

Serving (campaign recipe): `vllm serve /mnt/nas/home/ml/model/reranker/qwen3-reranker-4b
--runner pooling --hf_overrides '{"architectures":["Qwen3ForSequenceClassification"],
"classifier_from_token":["no","yes"],"is_original_qwen3_reranker":true}'` + the

## Wave-6 step dispositions

1. **Deterministic feature router** — built (`wave6_query.py` features + `route.*`
   channel + `wave6_fit_gate.py`); query-text features cannot route reranking (mining);
   retrieval-state features can (oracle +0.083 dev) but the fitted gate fails the
   repeated-cells slice gate and is **superseded by Result 3**. **REJECT as default.**
2. **Self-query soft filters** — **REJECT** (dev hit@5 −0.031; rank-fused soft design
   was safe, the extracted filters just misfire too often at 17% trigger).
3. **Query decomposition** — **REJECT** (LLM and deterministic both flat held-out;
   ungated variant regresses hit@5 −0.031).
4. **HyDE / multi-query** — **REJECT** (HyDE's dev +0.030 did not replicate and costs
   sentinel −0.027; MQ reaches R@20 1.000 without adding a single sentinel chunk).
   Identifier-protective gating (`no-ident`) was not the issue — `all` and gated fail
   alike.
5. **CRAG-style evaluator** — built, fitted, frozen, measured: dominates always-bge on
   every aggregate but inherits replicated slice damage via adverse selection;
   **REJECT as default**, retired along with the lever it optimized.
6. **Agentic loop** — **NOT RUN, closed by evidence**: static retrieval now sits at
   R@20 0.995 (code 1.000) with nDCG 0.848 after the substrate fix, and every routed
   mechanism this wave was dominated by that fix. A bounded agent can only re-query the
   same substrate; remaining headroom is answer-stage. Folded into **Wave 7** alongside
   the deferred 0019 packing arms and the two named caveat slices.

## The Wave-6 synthesis — one sentence

Adaptive query-time machinery (routers, gates, filters, transforms) was compensating
for a weak arbiter; serving the strong arbiter faithfully (instruction-templated
Qwen3-Reranker-4B) beats every adaptive mechanism it motivated, turns "rerank: lever or
liability?" into "always", and relocates the remaining open questions to the answer
stage.
